# ProcessOn 自动导出系统 - 运营使用说明书（Java后端版）

> 本文档面向运营人员，无需编程知识即可理解和使用。

---

## 一、系统能做什么？

这是一个**自动导出 ProcessOn 流程图**的机器人。你只需要告诉它：
- 要导出哪个流程图（链接）
- 用什么格式导出（PNG/JPG/PDF等）
- 图片品质要求（高清/普通）
- 是否需要水印

系统就会自动打开浏览器、登录、导出文件，并上传到阿里云 OSS，最后给你一个下载链接。

---

## 二、核心概念（用大白话解释）

### 1. 工作流（Workflow）

想象成一份**操作说明书**，告诉机器人每一步要做什么：
1. 打开网页
2. 输入密码
3. 点击导出按钮
4. 选择格式
5. 等待下载
6. 上传文件

工作流可以保存在两个地方：
- **配置文件**：`config/workflows/processon.yaml`（文件形式）
- **数据库**：通过 API 或 Web UI 管理（推荐，支持动态更新）

### 2. 变量（Variables）

工作流里有些信息是**动态的**，每次导出可能不一样，比如：
- 流程图链接（每次不同）
- 访问密码（每次不同）
- 导出格式（PNG/JPG/PDF）
- 品质要求（高清/普通）
- 水印设置（有/无）

这些动态信息用 `{{变量名}}` 表示，运行时会被替换成实际值。

### 3. 策略模式（Strategy）

不同导出格式的操作方式不一样：
- **PNG/JPG**：需要弹窗选择品质和水印
- **PDF/SVG**：直接下载，没有弹窗

系统会自动识别格式，选择正确的操作方式，你不需要关心细节。

## 三、配置文件说明

### 文件位置

```
config/
├── default.yaml          # 主配置文件（控制服务器、浏览器、定时任务等）
├── production.yaml       # 生产环境配置（上线时使用）
└── workflows/
    └── processon.yaml    # ProcessOn 导出工作流（运营主要关注这个）
```

### processon.yaml 工作流详解

```yaml
name: "Processon动态导出流程"
description: "支持动态参数传入的ProcessOn流程图导出工作流"

# ========== 变量定义 ==========
# 这些变量可以在运行时从外部传入
variables:
  # 基础配置
  baseUrl: "https://www.processon.com"
  
  # 动态参数（从外部传入）
  url: ""           # 流程图链接
  password: ""      # 访问密码（如果没有密码，可以不传）
  format: "jpg"     # 导出格式
  quality: "normal" # 品质：hd(高清) / normal(普通)
  watermark: "none" # 水印：system(系统) / none(无) / custom(自定义)
```

**变量说明：**

| 变量名 | 用途 | 示例值 |
|--------|------|--------|
| `url` | 流程图分享链接 | `https://www.processon.com/view/link/69c21af3570ada05a4ee6e6a` |
| `password` | 访问密码（如果有） | `hL2X` |
| `format` | 导出格式 | `jpg`, `png`, `pdf`, `svg` 等 |
| `quality` | 图片品质 | `hd`(高清), `normal`(普通) |
| `watermark` | 水印类型 | `system`(系统), `none`(无), `custom`(自定义) |

### 导出格式对照表

| 格式 | 说明 | 是否有预览弹窗 | 支持品质设置 | 支持水印设置 |
|------|------|---------------|-------------|-------------|
| `png` | 普通 PNG | 有 | ✓ | ✓ |
| `png-hd` | 高清 PNG | 有 | - | ✓ |
| `jpg` | 普通 JPG | 有 | ✓ | ✓ |
| `jpg-hd` | 高清 JPG | 有 | - | ✓ |
| `pdf` | 高清 PDF | 无 | - | - |
| `svg` | SVG 矢量图 | 无 | - | - |
| `pos` | POS 源文件 | 无 | - | - |
| `visio` | Visio 文件 | 无 | - | - |

**注意：**
- 带 `-hd` 的格式默认就是高清，不需要再设置 `quality`
- PDF/SVG/POS/VISIO 直接下载，没有品质和水印选项

## 四、日常使用场景

### 场景1：手动导出单个流程图

**使用命令行：**

```bash
# 导出 JPG，普通品质，系统水印
mvn spring-boot:run -Dspring-boot.run.arguments="run \
  --url https://www.processon.com/view/link/xxx \
  --password hL2X \
  --format jpg \
  --quality normal \
  --watermark system"

# 导出高清 PNG，无水印
mvn spring-boot:run -Dspring-boot.run.arguments="run \
  --url https://www.processon.com/view/link/xxx \
  --format png \
  --quality hd \
  --watermark none"

# 导出 PDF（直接下载）
mvn spring-boot:run -Dspring-boot.run.arguments="run \
  --url https://www.processon.com/view/link/xxx \
  --format pdf"
```

**使用 API：**

```bash
curl -X POST http://localhost:8080/api/trigger \
  -H "Content-Type: application/json" \
  -d '{
    "workflowId": "processon",
    "url": "https://www.processon.com/view/link/xxx",
    "password": "hL2X",
    "format": "jpg",
    "quality": "normal",
    "watermark": "system"
  }'
```

### 场景2：批量导出多个流程图

创建一个脚本文件 `batch-export.sh`：

```bash
#!/bin/bash

# 定义要导出的流程图列表
# 格式：链接|密码|格式|品质|水印
declare -a diagrams=(
  "https://www.processon.com/view/link/aaa|pass1|jpg|normal|system"
  "https://www.processon.com/view/link/bbb||png|hd|none"
  "https://www.processon.com/view/link/ccc|pass3|pdf||"
)

# 循环导出
for item in "${diagrams[@]}"; do
  IFS='|' read -r url password format quality watermark <<< "$item"
  
  echo "正在导出: $url"
  
  mvn spring-boot:run -Dspring-boot.run.arguments="run \
    --url $url \
    --password $password \
    --format $format \
    --quality $quality \
    --watermark $watermark"
  
  echo "完成: $url"
  echo "---"
done
```

执行：
```bash
chmod +x batch-export.sh
./batch-export.sh
```

### 场景3：定时自动导出

编辑 `config/default.yaml`：

```yaml
schedule:
  enabled: true        # 启用定时任务
  cron: "0 9 * * 1"   # 每周一上午9点执行
```

同时需要在工作流中指定默认的导出配置：

```yaml
variables:
  url: "https://www.processon.com/view/link/xxx"  # 默认链接
  password: "your-password"                        # 默认密码
  format: "jpg"
  quality: "normal"
  watermark: "system"
```

### 场景4：通过 Web UI 管理

Java 后端保留了前端页面，可以通过浏览器访问：

1. 启动后端服务：
   ```bash
   mvn spring-boot:run
   ```

2. 打开浏览器访问：
   - 首页：`http://localhost:8080`
   - 导出页面：`http://localhost:8080/export.html`

3. 在 Web UI 中可以：
   - 配置认证信息（Cookie/Token）
   - 查看工作流列表
   - 手动触发导出任务

## 五、常见问题

### Q1: 导出失败怎么办？

**检查清单：**
1. 链接是否正确？（必须是 `/view/link/` 格式的分享链接）
2. 密码是否正确？（注意大小写）
3. 认证信息是否过期？（需要重新在 Web UI 中更新 Cookie）
4. 格式参数是否拼写正确？（`jpg` 不是 `jpeg`）
5. 后端服务是否已启动？（检查 `http://localhost:8080/api/health`）

### Q2: 如何更新认证信息？

**方式1：通过 Web UI**
1. 打开 `http://localhost:8080`
2. 登录 ProcessOn 网站，按 F12 打开开发者工具
3. 复制 Cookie 字符串
4. 在 Web UI 中粘贴并保存

**方式2：通过 API**
```bash
curl -X PUT http://localhost:8080/api/auth \
  -H "Content-Type: application/json" \
  -d '{
    "cookies": [
      {
        "name": "session_id",
        "value": "your-session-value",
        "domain": ".processon.com"
      }
    ]
  }'
```

### Q3: 导出的文件在哪里？

- **本地：** `data/downloads/` 目录
- **OSS：** 登录阿里云控制台查看

### Q4: 如何修改默认导出配置？

编辑 `config/workflows/processon.yaml` 中的 `variables` 部分：

```yaml
variables:
  format: "png"       # 改成默认导出 PNG
  quality: "hd"       # 改成默认高清
  watermark: "none"   # 改成默认无水印
```

### Q5: 如何管理数据库中的工作流？

**查看所有工作流：**
```bash
curl http://localhost:8080/api/workflows
```

**查看指定工作流：**
```bash
curl http://localhost:8080/api/workflows/processon
```

**保存工作流到数据库：**
```bash
curl -X POST http://localhost:8080/api/workflows \
  -H "Content-Type: application/json" \
  -d '{
    "code": "processon",
    "name": "Processon动态导出流程",
    "description": "ProcessOn自动导出工作流",
    "content": "name: Processon动态导出流程\nsteps: [...]"
  }'
```

**删除工作流：**
```bash
curl -X DELETE http://localhost:8080/api/workflows/processon
```

## 六、配置速查表

### 命令行参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--url` | 流程图链接 | `--url https://www.processon.com/view/link/xxx` |
| `--password` | 访问密码 | `--password hL2X` |
| `--format` | 导出格式 | `--format jpg` |

| `--watermark` | 水印类型 | `--watermark system` |
| `--workflow` | 工作流文件路径 | `--workflow ./config/workflows/processon.yaml` |
| `--skip-delivery` | 跳过文件分发 | `--skip-delivery` |

### API 请求体

```json
{
  "workflowId": "processon",
  "url": "流程图链接",
  "password": "访问密码",
  "format": "导出格式",
  "quality": "hd 或 normal",
  "watermark": "system/none/custom"
}
```
