## CLI 命令

### 命令格式

```bash
# 使用 Maven
mvn spring-boot:run -Dspring-boot.run.arguments="<command> [options]"

# 或使用打包后的 JAR
java -jar target/auto-diagram-crawler-java-1.0.0.jar <command> [options]
```

### `run` - 执行导出流程

执行完整的导出工作流。

**基本用法：**

```bash
mvn spring-boot:run -Dspring-boot.run.arguments="run"
```

**常用选项：**

| 选项 | 简写 | 说明 | 示例 |
|------|------|------|------|
| `--workflow` | `-w` | 指定工作流文件路径 | `--workflow ./config/workflows/processon.yaml` |
| `--skip-delivery` | - | 跳过文件分发 | `--skip-delivery` |
| `--url` | - | 目标 URL | `--url "https://www.processon.com/view/link/xxx"` |
| `--password` | - | 访问密码 | `--password "hL2X"` |
| `--format` | - | 导出格式 | `--format "jpg"` |
| `--quality` | - | 图片品质 | `--quality "normal"` |
| `--watermark` | - | 水印类型 | `--watermark "system"` |

**示例：**

```bash
# 使用默认工作流执行
mvn spring-boot:run -Dspring-boot.run.arguments="run"

# 指定工作流文件
mvn spring-boot:run -Dspring-boot.run.arguments="run --workflow ./config/workflows/processon.yaml"

# 跳过文件分发（仅下载到本地）
mvn spring-boot:run -Dspring-boot.run.arguments="run --skip-delivery"

# 导出指定 URL 的流程图（JPG，普通品质，系统水印）
mvn spring-boot:run -Dspring-boot.run.arguments="run \
  --url https://www.processon.com/view/link/69c21af3570ada05a4ee6e6a \
  --password hL2X \
  --format jpg \
  --quality normal \
  --watermark system"

# 导出高清 PNG，无水印
mvn spring-boot:run -Dspring-boot.run.arguments="run \
  --url https://www.processon.com/view/link/xxx \
  --format png \
  --quality hd \
  --watermark none"

# 导出 PDF（直接下载，无需品质/水印参数）
mvn spring-boot:run -Dspring-boot.run.arguments="run \
  --url https://www.processon.com/view/link/xxx \
  --format pdf"

# 导出 SVG
mvn spring-boot:run -Dspring-boot.run.arguments="run \
  --url https://www.processon.com/view/link/xxx \
  --format svg"
```

### `validate` - 验证工作流配置

检查工作流 YAML 文件的语法和配置是否正确。

**用法：**

```bash
mvn spring-boot:run -Dspring-boot.run.arguments="validate"
mvn spring-boot:run -Dspring-boot.run.arguments="validate --workflow ./config/workflows/processon.yaml"
```

**输出示例：**

```

  Name: Processon动态导出流程
  Steps: 67

```

### GET /api/auth

获取当前保存的认证信息。

```bash
curl http://localhost:8080/api/auth
```

**响应：**

```json
{
  "cookies": [
    {

      "value": "xxx",
      "domain": ".processon.com"
    }
  ],
  "token": "your-token",
  "tokenHeader": "Authorization",
  "headers": {}
}
```

### PUT /api/auth

保存认证信息。

**请求：**

```bash
curl -X PUT http://localhost:8080/api/auth \
  -H "Content-Type: application/json" \
  -d '{
    "cookies": [
      {
        "name": "session_id",
        "value": "your-session-value",
        "domain": ".processon.com"
      }
    ],
    "token": "your-token",
    "tokenHeader": "Authorization",
    "headers": {}
  }'
```

**请求体字段：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `cookies` | array | 否 | Cookie 列表 |
| `token` | string | 否 | Token 值 |
| `tokenHeader` | string | 否 | Token 的请求头名称 |
| `headers` | object | 否 | 自定义请求头 |

### POST /api/trigger

手动触发一次导出任务。

**请求：**

```bash
curl -X POST http://localhost:8080/api/trigger \
  -H "Content-Type: application/json" \
  -d '{
    "workflowId": "processon",
    "url": "https://www.processon.com/view/link/xxx",
    "password": "hL2X",
    "format": "jpg",
    "quality": "normal",
    "watermark": "system"
  }'
```

**请求体字段：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `workflowId` | string | 否 | 工作流ID（从数据库加载，默认使用配置文件） |
| `url` | string | 否 | 目标流程图链接 |
| `password` | string | 否 | 访问密码 |
| `format` | string | 否 | 导出格式 |
| `quality` | string | 否 | 图片品质：`hd` / `normal` |
| `watermark` | string | 否 | 水印类型：`system` / `none` / `custom` |

**响应：**

```json
{
  "success": true,
  "downloads": [
    "/app/data/downloads/export_xxx.jpg"
  ],
  "delivery": [
    {
      "pluginName": "aliyun-oss",
      "success": true,
      "message": "File uploaded successfully",
      "url": "https://xxx.oss-cn-hangzhou.aliyuncs.com/exports/export_xxx.jpg"
    }
  ],
  "duration": 15432
}
```

**响应字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `success` | boolean | 工作流是否成功完成 |
| `downloads` | array | 下载到本地的文件路径列表 |
| `delivery` | array | 各分发插件的执行结果 |
| `duration` | number | 总耗时（毫秒） |

## 参数详解

### 导出格式（format）

| 值 | 说明 | 预览弹窗 | 支持品质 | 支持水印 |
|----|------|---------|---------|---------|
| `png` | 普通 PNG | 有 | ✓ | ✓ |
| `png-hd` | 高清 PNG | 有 | - | ✓ |
| `jpg` | 普通 JPG | 有 | ✓ | ✓ |
| `jpg-hd` | 高清 JPG | 有 | - | ✓ |
| `pdf` | 高清 PDF | 无 | - | - |
| `svg` | SVG 矢量图 | 无 | - | - |
| `pos` | POS 源文件 | 无 | - | - |
| `visio` | Visio 文件 | 无 | - | - |

### 图片品质（quality）

| 值 | 说明 | 适用格式 |
|----|------|---------|
| `hd` | 高清 | `png`, `jpg` |
| `normal` | 普通 | `png`, `jpg` |

**注意：** `png-hd` 和 `jpg-hd` 格式固定为高清，不需要设置此参数。

### 水印类型（watermark）

| 值 | 说明 | 适用格式 |
|----|------|---------|
| `system` | 系统水印 | `png`, `png-hd`, `jpg`, `jpg-hd` |
| `none` | 无水印 | `png`, `png-hd`, `jpg`, `jpg-hd` |
| `custom` | 自定义水印 | `png`, `png-hd`, `jpg`, `jpg-hd` |

## 使用示例

### 示例1：命令行导出不同格式

```bash
#!/bin/bash

# 定义流程图信息
URL="https://www.processon.com/view/link/69c21af3570ada05a4ee6e6a"

  --url $URL \
  --password $PASSWORD \

  --quality normal \
  --watermark system"

# 导出高清 PNG 无水印
echo "导出 PNG..."
mvn spring-boot:run -Dspring-boot.run.arguments="run \
  --url $URL \
  --password $PASSWORD \
  --format png \
  --quality hd \
  --watermark none"

# 导出 PDF
echo "导出 PDF..."
mvn spring-boot:run -Dspring-boot.run.arguments="run \
  --url $URL \
  --password $PASSWORD \
  --format pdf"
```

### 示例2：API 批量导出

```bash
#!/bin/bash

API_URL="http://localhost:8080/api/trigger"

# 导出配置列表
configs=(
  '{"url":"https://www.processon.com/view/link/aaa","password":"pass1","format":"jpg","quality":"normal","watermark":"system"}'
  '{"url":"https://www.processon.com/view/link/bbb","password":"","format":"png","quality":"hd","watermark":"none"}'
  '{"url":"https://www.processon.com/view/link/ccc","password":"pass3","format":"pdf"}'
)

for config in "${configs[@]}"; do
  echo "正在导出: $config"
  
  response=$(curl -s -X POST "$API_URL" \
    -H "Content-Type: application/json" \
    -d "$config")
  
  # 解析响应
  success=$(echo "$response" | grep -o '"success":true' | wc -l)
  
  if [ "$success" -gt 0 ]; then
    echo "✓ 导出成功"
    # 提取 OSS URL
    oss_url=$(echo "$response" | grep -o '"url":"[^"]*"' | head -1)
    echo "  文件地址: $oss_url"
  else
    echo "✗ 导出失败"
    echo "  响应: $response"
  fi
  
  echo "---"
done
```

### 示例3：Node.js 调用 API

```javascript
const axios = require('axios');

async function exportDiagram(config) {
  try {
    const response = await axios.post('http://localhost:8080/api/trigger', {
      workflowId: config.workflowId || 'processon',
      url: config.url,
      password: config.password,
      format: config.format || 'jpg',
      quality: config.quality || 'normal',
      watermark: config.watermark || 'none'
    });
    
    const result = response.data;
    
    if (result.success) {
      console.log('导出成功！');
      console.log('本地文件:', result.downloads);
      
      if (result.delivery && result.delivery.length > 0) {
        result.delivery.forEach(r => {
          if (r.success && r.url) {
            console.log('OSS 地址:', r.url);
          }
        });
      }
    } else {
      console.error('导出失败');
    }
    
    return result;
  } catch (error) {
    console.error('请求失败:', error.message);
    throw error;
  }
}

// 使用示例
exportDiagram({
  url: 'https://www.processon.com/view/link/xxx',
  password: 'hL2X',
  format: 'jpg',
  quality: 'normal',
  watermark: 'system'
});
```

### 示例4：Python 调用 API

```python
import requests
import json

def export_diagram(url, password='', format='jpg', quality='normal', watermark='none', workflow_id='processon'):
    """
    调用 ProcessOn 导出 API
    """
    api_url = 'http://localhost:8080/api/trigger'
    
    payload = {
        'workflowId': workflow_id,
        'url': url,
        'password': password,
        'format': format,
        'quality': quality,
        'watermark': watermark
    }
    
    try:
        response = requests.post(api_url, json=payload)
        result = response.json()
        
        if result['success']:
            print(f"✓ 导出成功")
            print(f"  本地文件: {result['downloads']}")
            
            if result['delivery']:
                for r in result['delivery']:
                    if r['success'] and 'url' in r:
                        print(f"  OSS 地址: {r['url']}")
            
            return result
        else:
            print(f"✗ 导出失败")
            return None
            
    except Exception as e:
        print(f"✗ 请求失败: {e}")
        return None

# 使用示例
export_diagram(
    url='https://www.processon.com/view/link/xxx',
    password='hL2X',
    format='jpg',
    quality='normal',
    watermark='system'
)
```

## 错误处理

### 常见错误码

| 错误 | 原因 | 解决方案 |
|------|------|---------|
| `EPERM` | 权限不足 | 检查文件/目录权限 |
| `ECONNREFUSED` | 服务未启动 | 启动 Web 服务 |
| `Timeout` | 操作超时 | 增加 timeout 参数 |
| `Validation Error` | 配置错误 | 检查 YAML 语法 |
| `Unknown step type` | 工作流步骤类型不支持 | 检查工作流配置 |

### 调试模式

开发调试时，建议：

1. **关闭无头模式**（查看浏览器窗口）：
   编辑 `config/default.yaml`：
   ```yaml
   browser:
     headless: false
     slowMo: 100  # 增加操作延迟
   ```

2. **跳过文件分发**：
   ```bash
   mvn spring-boot:run -Dspring-boot.run.arguments="run --skip-delivery"
   ```

3. **查看详细日志**：
   编辑 `src/main/resources/application.properties`：
   ```properties
   logging.level.com.exportbot.crawler=DEBUG
   ```

## 附录

### Cron 表达式参考

```
┌────────────── 分 (0-59)
│ ┌──────────── 时 (0-23)
│ │ ┌────────── 日 (1-31)
│ │ │ ┌──────── 月 (1-12)
│ │ │ │ ┌────── 星期几 (0-7, 0和7都是周日)
│ │ │ │ │
* * * * *
```

| 表达式 | 含义 |
|--------|------|
| `0 9 * * 1` | 每周一上午 9:00 |
| `0 9 * * *` | 每天上午 9:00 |
| `0 9,18 * * 1-5` | 工作日上午 9:00 和下午 6:00 |
| `*/30 * * * *` | 每 30 分钟 |
| `0 0 1 * *` | 每月 1 号 0:00 |

### 环境变量

| 变量 | 说明 |
|------|------|
| `SERVER_PORT` | Web 服务端口（默认 8080） |
| `SMTP_HOST` | SMTP 服务器地址 |
| `SMTP_PORT` | SMTP 端口 |
| `SMTP_USER` | SMTP 用户名 |
| `SMTP_PASS` | SMTP 密码 |
| `OSS_REGION` | OSS 区域 |
| `OSS_ACCESS_KEY_ID` | OSS Access Key |
| `OSS_ACCESS_KEY_SECRET` | OSS Secret |
| `OSS_BUCKET` | OSS Bucket 名称 |
